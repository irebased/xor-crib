XOR Crib Dragging Tool

Overview

This tool performs XOR crib dragging analysis on ciphertext that may have been rotated and/or XORed with a repeating key. It helps identify the correct transformation order and parameters by testing various configurations and highlighting bytes that match expected plaintext patterns.

How It Works

  1. Input Processing: The tool accepts ciphertext and an XOR key in multiple formats (base64, hex, decimal, octal, ASCII, binary).
  2. Cache Building: The XOR key is used to build a cache by XORing each key byte with common ASCII characters (A-Z, a-z, 0-9). This creates a lookup table of possible XOR result bytes.
  3. Matrix Transformation (optional): The ciphertext bits can be reorganized into a matrix of various dimensions and read in different patterns:
    • Standard: Read row by row (top-left to bottom-right)
    • Spin Right: Read columns from bottom to top (left to right)
    • Spin Left: Read columns from right to left (top to bottom)
  4. Rotation Analysis: The tool tests two scenarios:
    • Scenario 1: Rotated then XORed: Tests if the ciphertext was rotated first, then XORed
    • Scenario 2: XORed then Rotated: Tests if the ciphertext was XORed first, then rotated
    For each scenario, it tests rotations of 0-7 bits.
  5. Match Detection: After applying transformations, the tool checks each byte against the cache. If a byte matches a cached value, it means that byte could be the result of XORing a common ASCII character with the key.
  6. Results: Results show match percentages, and you can hover over highlighted bytes to see which plaintext characters could produce that byte value.

Features

  • Manual Mode: Select specific matrix dimensions, reading mode, and view results for each rotation
  • Auto Mode: Automatically tests all combinations and ranks them by match percentage
  • Matrix Dimensions: Automatically calculates all possible matrix dimensions based on bit count
  • Real-time Updates: Matrix dimensions update as you type
  • Reverse Text: Quickly reverse the ciphertext input
  • Interactive Highlights: Hover over highlighted bytes to see candidate plaintext characters

Usage Tips

  • Start with Auto Mode to quickly find the best configurations
  • Look for configurations with high match percentages (>10-20%)
  • Use the "View Details" button to see the full binary output with highlights
  • Try different matrix dimensions if the bit count has many factors
  • The tool works best when the plaintext contains common ASCII characters (letters and numbers)
